empirical questions (E-1 through E-10). Seven were answered from a single vault

evidence, and the Director can read a plain-English summary confirming it all.

list doesn't cover everything needed to confidently proceed with Phase 2 (rules)
and Phase 3 (production code). Key gaps include: targeted variable queries,
multi-configuration files, virtual component handling, and cancel behavior.

## Key Decisions From This Planning Session

1. **BOM traversal is shallow** — validate the transitioning file + its immediate
   children only. No recursive traversal. Subassemblies that already transitioned
   were already validated. This is the Director's domain rule.
2. **Parts are validated too** — standalone SLDPRT files have metadata and are
   effectively Level 0 with no children. Many rules apply to them.
3. **E-2 (virtual components) is must-have** — virtual components are common at
   EnerCorp, not rare. Corrects earlier incorrect assumption.
4. **E-7 and E-9 combine** — test cancel behavior and cancel UX together by
   modifying the diagnostic to actually block a transition during testing.
5. **Multi-config files need testing** — some files have 100+ configurations.
   When the file itself transitions, all configs must be validated. Performance
   implications are unknown.
6. **File extension filtering** — production add-in processes only .SLDPRT and
   .SLDASM. Skip everything else immediately.
7. **Previous closeout plan is superseded** — the scope expanded significantly
   based on this session's review.

## Tasks

This is multi-session work. Tasks are ordered by dependency — some can run in
parallel during a single test session once the diagnostic is redeployed.

### Task 1: Modify diagnostic add-in for expanded testing

- **What:** Add four capabilities to the diagnostic: (1) machine-name guard so
  it only runs on the Director's PC, (2) cancel mode that blocks transitions
  when a flag file is present (for E-7/E-9), (3) targeted variable query
  section that reads specific variables by name and compares to full enumeration
  (for E-11), (4) log all configurations found on the transitioning file, not
  just the ones with values (for E-18).
- **Type:** build
- **Why:** The current diagnostic can't test cancel behavior, targeted queries,
  or multi-config enumeration. Machine guard is a safety requirement from the
  server outage incident.
- **Done when:** Modified diagnostic builds successfully, machine guard is
  confirmed working, new sections appear in log output.

### Task 2: Redeploy diagnostic and run core test transitions

- **What:** Deploy modified diagnostic DLL to the vault. Director runs a series
  of test transitions to capture data for all open questions. Minimum test
  transitions needed:
  - A standalone SLDPRT (non-weldment) → answers E-12, E-16
  - The weldment SLDPRT (A3-1035707) → answers cut-list discrepancy
  - An assembly with virtual components → answers E-2
  - A file with many configurations → answers E-18
  - A multi-file batch transition (with cancel mode on) → answers E-7/E-9
  - A file with "Exclude from BOM" checked → answers E-5
  - Any assembly transition → re-confirms E-1, E-4, E-6, E-8, E-10 and
    adds E-11, E-13, E-14, E-15 data
- **Type:** investigate
- **Why:** These transitions generate the evidence needed to answer every
  remaining question.
- **Output to:** `.scaffold/investigations/YYYYMMDD-NN-expanded-diagnostic-analysis.md`
- **Done when:** Log entries captured for each test scenario listed above.

### Task 3: Analyze results and answer all E-questions

- **Why:** The Director needs to understand and verify every finding in plain

- **Output to:** Feeds into Task 5 (closeout summary)
- **Done when:** Director has reviewed and confirmed understanding of every
